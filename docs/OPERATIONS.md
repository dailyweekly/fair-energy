# 운영 매뉴얼 (Operations)

> 본 문서는 공정에너지 PoC 운영자(NGO 케이스워커·관리자)용 매뉴얼이다.
> 평상 운영, 인시던트 대응, 사용자 지원 절차를 정리한다.

---

| 역할 | 책임 |

| **개발팀** | 코드·정책 유지, 게이트 모니터링, 인시던트 대응 |

## 3. 사용자 지원 시나리오

### 시나리오 A — 사용자가 OCR 추출값이 잘못됐다고 신고
1. 사용자에게 페이지 4(추출값 확인)에서 직접 수정하도록 안내
2. 「확인」 체크 후 다시 점수 페이지로 진입하도록 안내
3. 반복 발생 시 페이지 3b(수동 입력) 모드 권장

### 시나리오 B — 사용자가 메시지 예시 복사 버튼이 안 보인다고 함
1. 안전체크 응답 확인 → HIGH 또는 VERY_HIGH 등급일 가능성
2. **HIGH**: 페이지 7 상단의 상담기관 안내 확인 + 「위 안내를 확인하였으며 메시지 예시를 검토하고자 합니다」 체크박스 클릭 안내
3. **VERY_HIGH**: 메시지 예시 자체가 차단된 상태. NGO 케이스워커 또는 법률구조공단(132)으로 상담 연계
4. NGO 케이스워커는 메시지 예시 노출을 강제 활성화하지 않는다 — 정책 05 §3 준수

### 시나리오 C — 사용자가 「임대인이 위법한 청구를 했다」를 메시지에 작성
1. 페이지 7의 자동 차단 안내문 확인 → 사용자에게 표현 수정 요청
2. 정책 01 §8.1 허용 의도(정산 재확인 / 자료 확인 / 산식 공개 / 검침일 확인)로 재작성 안내
3. 사용자에게 「본 서비스는 임대인의 위법성을 판단하지 않으며, 자료 확인 요청에 한정됩니다」 안내

### 시나리오 D — 사용자가 PDF에 임대인 이름이 들어가야 한다고 요청
1. 정책 04 §8 — 임대인 정보는 마스킹본·PDF에 포함되지 않는다
2. 대안 안내: 별도 동의 절차 후 비마스킹 사본을 사용자 본인이 받을 수 있음 (정책 04 §7)
3. NGO·외부 기관 제출은 건별·기관별 별도 동의 절차로 처리

### 시나리오 E — 사용자가 「개인정보 삭제」를 요청
1. 개보법 §36 사용자 권리 — 즉시 처리
2. 절차:
   ```sql
   -- 사용자의 documents·extracted_fields·passport_scores·calculation_results는 CASCADE로 함께 삭제
   DELETE FROM users WHERE user_id = ?;
   ```
3. 디스크의 `storage/originals/{user_id}/` 폴더도 삭제
4. 처리 결과를 사용자에게 회신
5. 처리 완료 후 audit_logs에 시스템 작업 기록

## 4. 인시던트 대응

### 인시던트 1 — 원본 파일 유출 의심
1. **즉시**: 영향 범위 파악 — `audit_logs`에서 시점·접근자 조회
2. 영향 사용자 통지
3. `APP_SECRET_KEY` 회전 (기존 키로 복호화 가능한 원본은 재암호화)
4. 사고 경위·대응 결과 보고서 작성 (개보위 신고 검토)
5. 정책 04 §9·§10 회귀 점검

### 인시던트 2 — 가드레일 우회 출력 발견
1. **즉시**: 우회 패턴을 `core/guardrails.FORBIDDEN_PATTERNS`에 추가
2. `tests/test_guardrails.py`에 해당 케이스 회귀 테스트 추가
3. 영향받은 사용자 화면에서 텍스트가 노출됐는지 audit_logs로 확인
4. 정책 01 §2 갱신

### 인시던트 3 — 룰 엔진 계산 오류
1. **즉시**: `ENABLE_LLM_OCR=false` + 모든 분류를 강제로 「검산 불가」로 변경 (운영 플래그)
2. tariff YAML 버전 확인 (정책 03 §2 메타데이터)
3. 한전 전기요금 전문가에게 재검수 의뢰
4. 영향 사용자에게 「검산 결과 재확인 필요」 통지

### 인시던트 4 — 사용자 보복 신고
1. NGO 케이스워커가 1차 대응
2. 폭언·협박·퇴거 압박이 있으면 즉시 법률구조공단(132)·여성긴급전화(1366) 안내
3. 사용자 데이터 즉시 보호 (외부 접근 차단)
4. NGO·법률구조공단과 협의 후 추가 대응

## 5. NGO 케이스워커 교육 핵심

### 안전체크 해석
- LOW: 일반 진행
- MEDIUM: 결과 확인 시 안전 안내 배너만 추가 노출 — 일반 진행
- **HIGH**: 메시지 예시 복사 버튼 차단됨. 사용자가 2단계 확인 후 진행 시에도 NGO 동석 권장.
- **VERY_HIGH**: 메시지 예시 본문 자체가 숨겨짐. **자료 정리만 진행하고 분쟁 행동 권유하지 않는다.** 주거상담기관·법률구조공단·여성긴급전화 우선 안내.

### 사용자에게 강조해야 할 메시지
> "공정에너지는 자료를 정리해서 본인이 검토하실 수 있게 돕는 도구입니다.
> 임대인에게 무엇을 보내야 하는지, 분쟁을 어떻게 진행해야 하는지는 판단하지 않습니다.
> 분쟁 단계로 가시려면 NGO나 법률구조공단의 도움을 먼저 받으세요."

### 절대 하지 말아야 할 것
- 사용자에게 「당신은 임대인이 부당하게 청구한 것입니다」라고 말하는 것
- 「소송하시면 이깁니다」 같은 결과 단정
- 사용자 동의 없이 임대인에게 자료 전달
- HIGH/VERY_HIGH 등급 사용자에게 메시지 예시 강제 활성화

## 6. 데이터 관리

### 보관기간 (정책 04 §6)
| 옵션 | 만료일 | 후처리 |
|---|---|---|
| 6개월 (기본) | created_at + 180일 | 원본 자동 파기, 마스킹본 유지 |
| 12개월 | created_at + 365일 | 동일 |
| 36개월 | created_at + 1095일 | 동일 |

만료 30일 전 사용자 알림 (재동의 절차 — 후속 구현 예정).

### 백업 정책
- DB (`fair_energy.db`): 매일 03:00 백업 → 30일 보관
- `storage/originals/`: 매일 03:30 백업 → 90일 보관 (별도 매체)
- `storage/masked/`·`storage/audit/`: 매주 백업
- 백업 시 암호화 키와 분리 저장

### 비식별·집계 통계 export
- `scripts/export_metrics.py`는 익명·집계 통계만 출력
- 개별 사용자 식별정보는 export 대상 아님
- NGO·심사위원·정부 제출용 보고서는 본 스크립트 결과만 사용

## 7. 정기 점검 체크리스트

```text
□ 만료 원본 자동 파기 cron 동작 확인
□ DB 백업 성공 확인 (yesterday's backup)
□ Streamlit 프로세스 정상 동작
□ KPI 대시보드 비정상 분포 없음
□ HIGH/VERY_HIGH 차단 이벤트 운영 보고
□ tariff YAML source URL 변경 여부 (분기)
□ 정책 문서 변경 사항 없음 (분기)
□ 다음 자문 일정 점검 (월간)
```

## 8. 자문·외부 협력 연락처

본 절은 운영자가 빠르게 연락할 수 있도록 협력 기관 연락처를 보관한다.
실제 연락처는 별도 secrets 파일로 관리하며 본 문서에는 카테고리만 표기한다.

- 화우 ESG센터 (pro bono·CSR 협력)
- 한전 전기요금 전문가 자문가
- 주거 NGO 협력 파트너 (PoC 동행)
- 외국인 노동자 지원기관 (다국어 대비)
- 법률구조공단 (분쟁 단계 직접 연결)
- 개인정보 외부 검토 자문가

연락처 변경 시 본 문서 갱신 + 백업 파일 동기화.
