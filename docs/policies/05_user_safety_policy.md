# 05. 사용자 안전 지침 (User Safety Policy)

> 본 문서는 사용자를 분쟁으로 떠밀지 않기 위한 안전 설계 기준이다.
> 신청서 v5 「리스크 및 대응 방안 4) 임차인 보복·퇴거 압박 리스크」와 일치한다.

---

## 1. 원칙

공정에너지의 대상은 **취약 임차인**이다. 고시원·반지하·외국인 노동자·청년 1인가구는 자료 요청 자체가 보복·퇴거 압박으로 이어질 수 있다.

> **검산 가능성보다 사용자 안전이 항상 우선한다.**

검산 결과가 「공식 산식 비교 가능」이어도, 안전 신호가 감지되면 자료요청 메시지 예시를 먼저 보여주지 않는다.

---

## 2. 안전 체크 문항 (Safety Check)

`pages/2_동의_및_안전체크.py`에서 동의 단계 직후 다음 6문항을 받는다.

```
1. 현재 계약갱신 또는 퇴거 협의 중인가요?
2. 보증금 반환이 걱정되나요?
3. 임대인으로부터 폭언, 협박, 퇴거 압박을 받은 적이 있나요?
4. 대체 주거지가 없나요?
5. 외국인 체류자격 또는 직장 제공 숙소와 연결되어 있나요?
6. 임대인에게 자료 요청을 보내는 것이 위험하다고 느끼나요?
```

각 문항은 예/아니오로 받는다. 각 문항을 위험 신호로 매핑한다.

---

## 3. 위험 신호 분류 (2026-05-13 외부 자문 반영)

`core/safety.py`에서 다음 4단계로 분류한다.

| 등급 | 조건 | 동작 |
|------|------|------|
| LOW | 모든 문항 「아니오」 | 일반 플로우 진행 |
| MEDIUM | 1~2개 「예」 (3·6번 제외) | 결과 화면에 안전 안내 배너 표시 |
| HIGH | 3번 또는 6번 단독 「예」, 또는 3개 「예」(3번 미포함) | 메시지 본문은 표시되되 「복사」 버튼은 비활성화. 주거상담기관 안내 우선. |
| **VERY_HIGH** | 3번(폭언·협박) + 4·5·6번 중 1개 이상, 또는 4개 이상 「예」 | **메시지 예시 본문 자체를 숨김**. 주거상담기관·법률구조공단·여성긴급전화 우선 안내. |

3번(폭언·협박·퇴거 압박)과 6번(자료 요청이 위험하다고 느낌)은 **자기보고 기반 직접적 위험 신호**다. 둘 중 하나만 「예」여도 HIGH 이상으로 분류한다.

**VERY_HIGH 신설 이유 (2026-05-13)**: 「HIGH 등급에서도 사용자가 메시지를 보내겠다고 할 때 막을 권한이 있는가」라는 자문 질의에 대해, 폭언·협박을 받은 상태에서 대체 주거가 없거나 외국인 신분 또는 본인이 자료 요청을 위험하다고 느끼는 경우는 **메시지 예시 자체를 노출하지 않는 것이 해악 방지(harm prevention) 설계**라는 권고를 반영.

---

## 4. HIGH 등급 시 대체 플로우

HIGH 등급 사용자에게는 검산 결과를 보여주되 자료요청 메시지 예시는 자동으로 표시하지 않는다. 대신 다음 문구를 결과 페이지 최상단에 표시한다.

```
안전 확인 필요

현재 상황에서는 임대인에게 직접 메시지를 보내기 전에
주거상담기관 또는 법률구조기관과 먼저 상담하는 것이 안전할 수 있습니다.

공정에너지는 자료 정리와 검산 가능성 확인만 도와드리며,
분쟁 행동을 권유하지 않습니다.

도움 받을 수 있는 곳:
  • 주거상담기관 (지자체 주거복지센터)
  • 법률구조공단 국번없이 132
  • 외국인 노동자 지원기관
  • 화우 ESG센터 공익 연계
```

사용자가 「그래도 메시지 예시 보기」 버튼을 누르면 메시지 예시를 제공하되, 상단에 위 안내가 계속 노출된다.

---

## 5. 자료요청 메시지의 안전 원칙

`pages/7_자료요청메시지.py`는 다음을 반드시 준수한다.

- 메시지는 **자동 작성이 아니라 예시**로 표시한다.
- 사용자가 **직접 편집**해야 함을 안내한다.
- 「복사」 버튼만 제공한다. **「임대인에게 자동 발송」 버튼은 금지**한다.
- 메시지 내용에는 「위반」·「부당청구」·「환급」 등 금지어가 포함되지 않는다(가드레일 통과).
- 발송 전 안전 체크를 재확인할 수 있는 링크를 메시지 화면에도 다시 노출한다.

---

## 6. 외국인·이주노동자 추가 보호

5번 문항이 「예」인 경우(외국인 체류자격 또는 직장 제공 숙소):

- 추가 안내 문구를 표시한다: 직장 제공 숙소의 경우 자료 요청이 고용 관계에 영향을 줄 수 있음.
- 다국어 안내(베트남·네팔어 등)는 2027년 후속 단계.
- PoC 단계에서는 외국인 노동자 지원기관 연락처를 별도 카드로 노출.

---

## 7. 시스템 책임의 한계

- 본 솔루션은 **위험 신호 자동 감지를 보장하지 않는다**. 사용자가 자기보고한 안전 체크 응답을 기준으로 한다.
- 사용자가 안전 체크를 정직하게 답하지 않거나 안내를 따르지 않은 결과로 발생한 피해는 본 솔루션이 책임지지 않는다.
- 위 한계는 면책 문구에 명시한다.

---

## 8. 운영자 모니터링

- HIGH 등급 케이스가 자료요청 메시지 「복사」 버튼을 누른 이벤트는 별도 로그로 남긴다(`audit_log` event_name=`high_risk_template_copied`).
- 분기별로 HIGH 케이스의 행동 패턴을 검토하여 안내 문구를 보강한다.

---

## 9. 테스트 기준

`tests/test_safety.py`는 다음을 반드시 통과해야 한다.

- 모든 문항 False → LOW
- 1번만 True → MEDIUM
- 1·2번 True → MEDIUM
- 3번 True (단독) → HIGH
- 6번 True (단독) → HIGH
- 1·2·4번 True (3개 이상, 3·6 미포함) → HIGH
- **3·4번 True (폭언 + 대체 주거 없음) → VERY_HIGH** (2026-05-13)
- **3·6번 True (폭언 + 자료 요청 위험감) → VERY_HIGH** (2026-05-13)
- **4개 이상 True → VERY_HIGH** (2026-05-13)
- HIGH 등급 → `should_block_message_copy()` True, `should_hide_message_template()` False
- **VERY_HIGH 등급 → `should_hide_message_template()` True** (2026-05-13)
- HIGH 등급 → `recommended_disclaimer()`에 「주거상담기관」 텍스트 포함
- VERY_HIGH 등급 → `recommended_disclaimer()`에 「긴급 안전 확인 필요」 텍스트 포함
