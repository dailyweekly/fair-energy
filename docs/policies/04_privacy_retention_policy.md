# 04. 개인정보·보관 지침 (Privacy & Retention Policy)

> 본 문서는 임차인·임대인 양측 개인정보, 원본/마스킹본 분리, 보관기간, 외부 제출 동의를 규정한다.
> 신청서 v5 「(3) AI 활용 설명 — 활용 데이터」, 「리스크 및 대응 방안 5)」와 일치한다.

---

## 1. 원칙

- 원본과 표시용 사본을 **분리** 저장한다.
- 원본은 **암호화** 저장한다.
- 표시용 사본은 **개인정보를 마스킹**한다.
- **임대인 정보도 개인정보**다. 임차인이 업로드했다고 해서 임대인 개인정보 처리 리스크가 사라지지 않는다.
- 보관기간은 사용자가 **선택**한다(6/12/36개월). 만료 전 재동의 없으면 원본은 자동 파기.
- 외부기관 제출은 **건별·기관별 별도 동의**를 받는다. 일괄 동의로 처리하지 않는다.
- **주민등록번호는 가능한 한 수집을 회피**한다. 처리 불가피 시 개인정보보호법 24조의2 법령상 근거를 사전 확인한다.

## 4. 마스킹 규칙

`core/masking.py`에 단일 구현한다. 패턴별 함수와 통합 `mask_text()` 함수 제공.

| 항목 | 예시 입력 | 마스킹 결과 |
|------|-----------|-------------|
| 이름 (한글 2자) | 홍길 | 홍* |
| 이름 (한글 3자) | 홍길동 | 홍** |
| 이름 (한글 4자 이상) | 홍길동민 | 홍*** |
| 전화번호 | 010-1234-5678 | 010-****-5678 |
| 계좌번호 | 123-45-67890 | 123-**-***** |
| 주민등록번호 | 900101-1234567 | ******-******* |
| 주소 (시·구 이후) | 서울시 관악구 봉천동 123-45 | 서울시 관악구 *** |
| 이메일 | abc@example.com | a**@example.com |

마스킹은 **항상 표시용 사본에만 적용**한다. 원본은 그대로 암호화 저장한다.

---

## 5. 저장 구조

```
storage/
├── originals/                 # 암호화 원본
│   └── {user_id}/{document_id}.bin
├── masked/                    # 마스킹된 표시용 사본
│   └── {user_id}/{document_id}.md  (2026-05-13: 텍스트 요약본)
└── audit/                     # 접근 로그
    └── {YYYY-MM}.log
```

- `originals/`는 `cryptography.Fernet`로 암호화한다.
- 암호화 키는 환경변수 `APP_SECRET_KEY`에서 로드하며, 32-byte 키여야 한다.
- `masked/`는 평문이지만 마스킹이 완료된 파일이다.

### 시각 마스킹 방침 (2026-05-13 외부 자문 반영)

**보수적 접근 — 원본 시각 정보를 마스킹본에 포함하지 않는다.**

원본 이미지·PDF를 픽셀 단위로 redact 하지 않는다(OCR 좌표 부재·false negative 위험).
대신 사용자가 직접 확인한 추출 필드만으로 **「텍스트 요약본」 (Markdown)** 을 재생성한다.
이 요약본이 `documents.masked_path`에 저장된다.

- 구현: [core/redaction.py](../../core/redaction.py)의 `regenerate_masked_view()`.
- 포함되는 정보: `key`, `label`, `value`, `unit` (사용자 확인된 필드만).
- 포함되지 않는 정보: `source_text` (원문 인용), 원본 이미지, 임대인 서명/도장/계좌.
- 수동 입력 문서(`original_path IS NULL`)는 별도 마스킹본을 생성하지 않는다.

**대안 (현재 MVP 미적용)**: `heavy_blur_image()` — 계량기 사진처럼 시각 자체가 본질인
경우의 전체 가우시안 블러 폴백. Phase 9 이후 필요 시 도입.

## 6. 보관기간

| 옵션 | 기본 만료일 | 만료 후 처리 |
|------|-------------|--------------|
| 6개월 (기본) | created_at + 180일 | 원본 자동 파기, 마스킹본은 유지 옵션 |
| 12개월 | created_at + 365일 | 동일 |
| 36개월 | created_at + 1095일 | 동일 |

- 만료 30일 전 사용자 알림.
- 만료일 도래 시 재동의 없으면 원본은 자동 삭제(`core/storage.py`의 cleanup job).
- 통계 분석에는 비식별·집계 데이터만 사용한다(가명처리, 법 28조의2).

---

## 7. 외부 제출 동의 (건별·기관별)

분쟁기관(주택임대차분쟁조정위·한전 전기소비자보호센터 등) 제출이 필요한 경우:

1. **제출 대상 기관** 명시
2. **제출 문서 범위** 명시
3. **제출 목적** 명시
4. 위 3가지를 사용자에게 별도로 고지하고 동의 받기
5. 동의 받은 뒤에만 원본 또는 비마스킹 사본 제공
6. 일괄 동의로 처리하지 않음

---

## 8. 임대인 정보 처리 (특별 주의)

- 임대인 식별정보(이름·계좌·전화·주소)는 표시용 사본에서 **별도 마스킹**한다.
- 임대인 정보의 **제3자 제공을 절대 금지**한다.
- NGO 어드민에는 **익명·집계 통계**만 노출한다(개별 임대인 식별 불가).
- 동 단위·주거유형 단위 **10건 이상**일 때만 통계 표시(소수 식별 방지).
- 분쟁 단계에서 임대인 정보 활용 시 사용자에게 별도 고지·동의를 받는다.

---

## 9. 주민등록번호 (고유식별정보) 특별 처리 (2026-05-13 외부 자문 반영)

- 가능한 한 **수집을 회피**한다.
- **업로드 자체를 거부하는 것이 PoC 단계 안전 기본값**. 마스킹 후 저장보다 우선.
  - 텍스트 추출 가능한 파일(.txt·.md·.csv 등): `storage.save_document(scan_text_for_rrn=True)`로 사전 스캔하여 감지 시 `RRNDetectedError` 발생.
  - 이미지·PDF: OCR 단계의 `pii_detected[type='resident_registration_number']`로 감지 후 사용자에게 재업로드 안내.
- 거부 시 사용자에게 정확히 안내: **「주민등록번호로 보이는 패턴이 감지되었습니다. 해당 부분을 가린 후 다시 업로드해 주세요.」**
- 처리가 불가피한 경우 개인정보보호법 24조의2(주민등록번호 처리 제한)의 **법령상 근거를 사전에 확인**하고, 별도 동의를 받은 뒤 마스킹·암호화 저장한다.
- 본 결정은 자문 항목 C3(외부 검토)에서 최종 확정한다.

## 10. 접근 로그

`core/audit_log.py`로 단일 구현. 다음을 기록한다.

```
{
  "timestamp": ISO8601,
  "actor": user_id | system | ngo_worker | admin,
  "action": upload | view | download | delete | export,
  "object": document_id,
  "object_type": original | masked,
  "consent_basis": consent_id (외부 제출 시)
}
```

원본 접근(`object_type=original`)은 **사용자 본인 재동의 이벤트**가 있어야만 허용한다.

## 11. 사용자 권리 (개인정보보호법 35~37조)

다음 기능을 UI에서 제공한다.

- 개인정보 열람 요청

- 처리정지 요청
- 동의 철회

이 기능은 후속 단계에서 별도 페이지로 노출할 수 있으나, MVP에서는 최소한 「내 자료 삭제」 버튼만 제공한다.

- 이메일 마스킹
