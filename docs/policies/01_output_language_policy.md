# 01. 출력 표현 지침 (Output Language Policy)

> 본 문서는 사용자에게 노출되는 모든 텍스트의 표현 기준이다.
> LLM 출력, UI 라벨, PDF 리포트, 시스템 메시지, 알림 등 사용자 경로의 모든 텍스트가 본 문서의 규칙을 통과해야 한다.

---

## 1. 원칙

공정에너지의 모든 출력은 **사실 분류**로만 표현한다.
공정에너지의 모든 출력은 **단정 표현**을 사용하지 않는다.

세 가지 사실 분류만 허용한다.

1. **산식 일치** — 사용자 청구액과 공식 산식 계산값 사이에 의미 있는 차이가 확인되지 않음
2. **차이 발생** — 사용자 청구액과 공식 산식 계산값 사이에 차이가 확인됨 (위법성 판단 아님)
3. **근거 부족** — 공식 산식과 비교하기 위한 자료가 부족하여 검산을 수행하지 못함

신청서 v5 「(2) 솔루션 개요 — 명시적 비목표」와 일치한다.

---

## 2. 금지 표현 (Forbidden Expressions)

다음 표현은 사용자 화면, LLM 출력, PDF 리포트, 알림, 에러 메시지 어디에도 출력되면 안 된다.

| 금지어 | 금지 이유 |
|--------|-----------|
| 위반 | 위법성 판단 |
| 불법 | 위법성 판단 |
| 부당청구 / 부당 부과 | 부당성 단정 (정보통신망법 70조 명예훼손 소지) |
| 사기 | 형사책임 암시 |
| 가스라이팅 | 단정 표현 (명예훼손 소지) |
| 환급 가능 / 환급받을 수 있 | 환급 결과 단정 |
| 돌려받을 수 있 | 결과 단정 |
| 받을 수 있습니다 | 결과 단정 |
| 승소 / 승소 가능성 | 소송 결과 판단 |
| 고소 / 고발 | 형사절차 유도 |
| 신고하세요 / 신고하 | 행동 단정 |
| 소송하세요 / 소송 | 분쟁 행동 단정 |
| 소장 | 법률문서 작성 리스크 |
| 내용증명 | 법률문서 작성 리스크 |
| 법적으로 문제 | 법률평가 표현 |
| 임대인이 잘못 | 책임 단정 |
| 임대인이 위법 | 위법성 단정 |
| 로봇 변호사 | DoNotPay FTC 제재 사례 회피 |
| 과다 청구 / 과다청구 | 부당성 단정(2026-05-13 자문 반영) |
| 반환 요구 / 반환해 | 분쟁 행동 유도(2026-05-13) |
| 법적 조치 / 법적조치 | 분쟁 행동 단정(2026-05-13) |
| 보상 가능 / 보상받을 수 있 | 결과 단정(2026-05-13) |
| 부당 이득 / 부당이득 | 형사·민사 책임 단정(2026-05-13) |

위 패턴 검출은 `core/guardrails.py`의 `FORBIDDEN_PATTERNS`로 구현하며,
`tests/test_guardrails.py`에서 모든 패턴이 차단되는지 검증한다.

## 3. 허용 표현 (Allowed Expressions)

다음 표현만 결과 라벨로 사용한다.

| 분류 | 허용 표현 |
|------|-----------|
| 산식 일치 | 공식 산식과 큰 차이가 확인되지 않았습니다 |
| 차이 발생 | 공식 산식 기준 계산값과 사용자 청구액 사이에 차이가 확인됩니다 |
| 근거 부족 | 검산에 필요한 근거 자료가 부족합니다 |
| 사용자 미확인 | AI 추출값에 사용자 확인이 필요합니다 |
| 검산 불가 | 현재 자료만으로는 검산할 수 없습니다 |
| 정산 재확인 필요 | 정산 재확인이 필요할 수 있습니다 |
| 자료 부족 | 추가 자료가 필요합니다 |
| 케이스 분류 | 검산 불가 / 근거 요청 필요 / 예비 검산 가능 / 공식 산식 비교 가능 / 정액 관리비 케이스 |

「산식 차이」는 「위반」이 아니다. 사용자가 받은 정보가 공식 요금표와 다른 산식으로 계산되었다는 **수치 비교 결과**일 뿐이다.

## 4. 기본 면책 문구 (Default Disclaimer)

모든 결과 페이지 하단, 모든 PDF 리포트, 모든 메시지 예시 하단에 다음 문구를 표시한다.

```
본 서비스는 법률 자문, 법률 의견, 소송 전략, 환급 가능성 판단을 제공하지 않습니다.
본 결과는 사용자가 업로드하고 직접 확인한 자료를 바탕으로 한 요금 산식 검토 보조 자료입니다.
임대인의 위법성 여부, 환급 가능성, 분쟁 결과는 판단하지 않습니다.
분쟁이 예상되거나 계약갱신, 보증금 반환, 퇴거 압박, 신변 위험이 있는 경우
주거상담기관 또는 변호사 등 전문가와 먼저 상담하세요.
```

면책 문구는 `core/safety.py`의 `DEFAULT_DISCLAIMER` 상수로 단일화한다.
페이지마다 손으로 적지 않는다.

## 5. LLM 시스템 프롬프트 의무 문구

OCR/필드 추출용 LLM 시스템 프롬프트에는 다음을 반드시 포함한다.

```
1. 법률 판단을 하지 마세요.
2. 위법, 부당, 환급 가능성, 승소 가능성, 신고 필요성을 판단하지 마세요.
3. 문서에 없는 정보는 추측하지 마세요.
4. 사실 정보만 추출하세요.
5. 출력은 반드시 JSON만 반환하세요.
```

## 6. 가드레일 적용 의무 (Round 4 보강)

### 6.1 적용 대상

다음 텍스트는 모두 `core/guardrails`의 가드레일을 통과해야 한다.

- LLM 응답 후처리 → `assert_safe_output()` (엄격 모드)
- 메시지 템플릿 렌더링 → `assert_safe_output()` (엄격 모드)
- PDF 리포트·자료 요약본 → `assert_safe_output()` (엄격 모드)
- 마스킹본(`core/redaction.build_text_summary`) → `assert_safe_output()` (엄격 모드)
- 분류 디스크립터(`core/classification.CASE_DESCRIPTORS`) → `assert_safe_output()` (엄격 모드)
- 점수 액션(`core/scoring._build_actions`) → `assert_safe_output()` (엄격 모드)
- Streamlit UI 동적 텍스트 → `assert_safe_output()` (엄격 모드)

### 6.2 완화 모드 (Relaxed)

정적 면책 문구·정책 인용 텍스트는 `is_safe_output_relaxed()`/`assert_safe_output_relaxed()`로 검증한다.

- `core/safety.DEFAULT_DISCLAIMER`, `HIGH_RISK_DISCLAIMER`, `VERY_HIGH_RISK_DISCLAIMER`
- 정책 문서 인용 표현 ("환급 가능성 판단을 제공하지 않습니다" 등)

화이트리스트(`STATIC_SAFE_DISCLAIMER_PHRASES`)에 있는 phrase는 검사 전 제거되고 나머지에 엄격 가드레일이 적용된다.

화이트리스트 추가 조건:
1. 한 phrase 단위로 어휘적으로 일치해야 함 (변형 금지)
2. 부정/제한 문맥 안에서만 의미를 가져야 함
3. **그 자체가 엄격 모드에서 차단되어야 함** (가드레일 우회 방지)
4. 정적 상수 또는 정책 문서 인용 텍스트에만 사용

### 6.3 면제 대상

LLM 입력 프롬프트는 가드레일 검증 대상에서 면제된다.

- `core/prompts.OCR_SYSTEM_PROMPT` — LLM에게 금지어를 지시하기 위해 인용해야 함
- 프롬프트의 안전성은 별도 회귀(`tests/test_prompts.py`)로 보장 — 필수 문구 포함 여부 검증

### 6.4 사용자 입력 콘텐츠

사용자가 자유 입력한 필드 값(예: 수동 입력의 임의 메모, 임의로 수정한 OCR 추출값)은 가드레일을 적용하지 않는다. 사용자 입력은 본인 책임 영역이며, 시스템 생성 텍스트와 구분된다.

다만 사용자 입력이 외부에 노출되는 출력물(예: 마스킹본 PDF)에 그대로 들어가는 경우, 시스템 생성 부분(헤더·면책·라벨)에만 가드레일이 적용된다.

### 6.5 위반 처리

가드레일 위반은 `UnsafeOutputError`를 발생시키며, 화면 출력 전에 차단된다. catch 없이 전파시켜 개발자가 즉시 인지하도록 한다.

## 8. 메시지 표현 범위 (2026-05-13 외부 자문 반영)

빈칸형 자료요청 메시지 예시(Phase 9)에서 사용자가 임대인에게 보내는 문구의 **표현 범위**는 본 절에 의해 제한된다.

### 8.1 허용 표현 (Allowed Phrasing)

다음 의도만 메시지에 담길 수 있다.

- **정산 재확인 요청** — "정산 내역을 확인하고 싶습니다"
- **자료 확인 요청** — "전기요금 고지서 사본을 공유해 주실 수 있을까요"
- **산식 공개 요청** — "세대별 배분 산식을 알려주실 수 있을까요"
- **계량기 검침일 확인** — "검침일 기준이 어떻게 되는지 문의드립니다"

### 8.2 금지 표현 (Forbidden Phrasing)

다음 의도는 메시지에 절대 포함될 수 없다.

- **금액 단정** — "X원 과다하게 청구되었습니다" 금지
- **반환 요구** — "X원을 반환해 주십시오" 금지
- **법적 조치 암시** — "법적 조치를 검토 중입니다" 금지
- **분쟁 행동 예고** — "민원을 제기할 예정입니다" 금지
- **위법성 단정** — "위법한 청구입니다" 금지
- **임대인 신원 비난** — "임대인께서 잘못하셨습니다" 금지

### 8.3 메시지 운영 원칙 (Phase 9 R1 반영)

- 메시지는 **자동 작성이 아닌 「예시」**로 표시된다.
- 사용자가 **직접 편집**한다는 안내가 메시지 위·아래에 명시된다.
- **자동 채움 범위 제한**: 메시지 본문의 자동 채움은 `{address}`(주소·호실)와 `{billing_period}`(청구 기간) 2종만 허용된다. 청구액·임대인 이름·계좌·전화번호는 절대 자동 채우지 않는다.
- 「복사」 버튼만 제공된다. **자동 발송·자동 작성·자동 완성** 기능은 금지된다.
- 안전 등급별 차등 (정책 05 §3과 일치):
  - LOW: 표시 가능
  - MEDIUM: 경고 후 표시
  - HIGH: 상담기관 안내 우선 표시, 2단계 확인 후 메시지 노출
  - VERY_HIGH: 메시지 본문 자체 숨김, 자료 요약본만 제공
- 사용자가 메시지를 편집한 뒤에도 `validate_edited_message()`가 자동 차단 패턴을 검출하면 「복사」 버튼이 비활성화된다.
- 본 절의 분류(허용/금지)는 화우 ESG센터 사전 자문 의견서(G5) 수령 후 최종 확정된다.
  - 현재(2026-05-13)는 외부 전문가 추정 의견 기반의 「추정 안전 영역」.
  - 의견서 수령 후 표현이 확장 또는 축소될 수 있다.

### 8.4 메시지 명칭

문서명·UI 라벨에서 다음 표기를 강제한다.

- **권장**: 「자료 확인 메시지 예시」, 「자료 요청 메시지 예시」
- **금지**: 「자료요청서」, 「요청서」, 「내용증명」, 「민원」

### 8.5 분쟁 기관 절차 안내

- **정적 markdown 정보 카드** (`data/procedures/`)로만 제공된다.
- 개별 사안에 대한 「당신의 경우 X기관에 신청하세요」 식 추천 금지.
- 각 안내문에는 다음 문구를 명시한다: **"본 서비스는 어느 기관에 신청해야 하는지 판단하지 않습니다."**
- RAG 도입은 MVP 범위 외(정책 07 §2 G1·G2 N/A 결정).
