# 00. 서비스 범위 지침 (Service Scope Policy)

> 본 문서는 공정에너지(Fair Energy) — Energy Bill Passport MVP 개발의 최상위 정책 문서다.
> 본 문서와 충돌하는 다른 모든 문서·코드·UI는 본 문서를 따른다.

---

## 0. 일행 정체성 선언

공정에너지는 법률서비스가 아니다.
공정에너지는 임차인의 에너지 청구 자료가 검산 가능한 상태인지 진단하고,
공식 요금 산식과 비교 가능한 경우에만 계산 근거를 제공하는
**Energy Bill Passport 인프라**다.

이 한 문장이 흔들리면 변호사법 제109조 리스크가 즉시 발생한다.
모든 기능·UI·LLM 프롬프트는 이 선언과 충돌하면 안 된다.

---

## 1. 1차 MVP 범위 고정 (Scope Lock)

| 항목 | 범위 |
|------|------|
| 대상 | 서울·수도권에서 **한전이 아닌 임대인이 매달 전기요금을 분배 청구**하는 구조의 임차인. **기준은 주거 형태가 아니라 청구 구조.** 자주 발견되는 사례는 고시원·다가구 임차인이며, 다세대·반지하·원룸 등은 같은 건물·층이라도 호실마다 청구 구조가 달라 일률적으로 포함·배제하지 않는다. 한전이 본인 명의로 직접 청구서를 발부하고 임차인이 본인 명의 한전 고객번호를 가진 경우는 제외 — 한전 약관·요금표가 그대로 적용되어 별도 검산이 거의 필요하지 않음. |
| 영역 | 전기요금만 |
| 채널 | Streamlit 웹앱 (카카오톡 챗봇은 후속) |
| 인프라 | SQLite (이후 Supabase/PostgreSQL 확장) |
| AI | 멀티모달 LLM은 문서 구조화·필드 추출까지만 |
| 계산 | LLM 금지. `core/tariff_engine.py` 룰 엔진으로만 |

PoC 4개월(8~10월) 안에 위 범위를 벗어나면 변호사법 리스크와 OCR 정확도 리스크가 동시에 폭발한다.

## 2. 허용 기능 (Allowed Features)

다음 기능만 MVP에서 구현한다.

1. 사용자 동의 및 안전 체크
2. 문서 업로드 (계약서·관리비/전기료 청구·계량기 사진·한전 고지서·납부내역)
3. 멀티모달 OCR 및 필드 추출 (사실 정보만) — **AI 처리 미동의 시 수동 입력 모드로 대체 가능**
4. 사용자 추출값 직접 확인·수정
5. Energy Bill Completeness Score (100점 만점, 6개 자료별 배점)
6. 검산 가능성 4분류 (검산 불가 / 근거 요청 필요 / 예비 검산 가능 / 공식 산식 비교 가능)
7. 한전 전기요금 룰 엔진 계산
8. 산식 차이 표시 (중립 표현)
9. 빈칸형 자료요청 메시지 예시 (사용자 직접 편집)
10. 공개기관 절차 안내 (정적 markdown 기반 정보 카드 — **RAG 미사용**)
11. **에너지 청구 자료 요약본** PDF 다운로드 (시각 마스킹 적용 후만 활성화)
12. KPI 측정용 이벤트 로그

### 명명 규약
- **「증거 패키지」 표현 사용 금지.** 분쟁용 문서로 오인될 위험. 항상 **「에너지 청구 자료 요약본」**으로 표기.
- **RAG는 MVP 범위 외.** 기관 절차 안내는 정적 markdown 정보 카드로 제공.

## 3. 금지 기능 (Forbidden Features)

다음 기능은 MVP에서 절대 구현하지 않는다. 후속 단계에서도 화우 ESG센터 변호사법 사전 자문 의견서 없이는 도입하지 않는다.

- 위법성 판단 (위반·불법 여부 판단)
- 부당청구·사기 여부 판단
- 환급 가능액 산정
- 승소 가능성 판단
- 소장 작성
- 내용증명 작성
- 민원 본문 자동작성
- AI가 사용자 입력을 바탕으로 한 생성형 법률문서 작성
- 임대인에게 자동 발송
- 임대인 식별정보의 외부 공개
- 임대인 블랙리스트
- 건물명 공개 통계
- 「로봇 변호사」류 PR 표현 (DoNotPay FTC 제재 사례 회피)
- AI가 사건 검토를 수행한 뒤 변호사에게 넘기는 구조 (로폼 판결 「제휴 변호사 사건검토 서비스」 모방 금지)

---

## 4. 후속 단계로 명시 이연 (Deferred)

다음 기능은 2027년 이후 후속 고도화 단계로 명시 이연한다. PoC 4개월 안에는 절대 구현하지 않는다.

- 카카오톡 챗봇
- 도시가스·수도 요금 검산
- 관리비 전체 검산
- 다국어 (베트남·네팔·우즈베크어)
- 마이데이터 연동
- B2G 어드민 본격 기능
- 임대인 응답 자동 분석
- 사용자 간 사례 공유 기능
- **RAG 기반 법령·기관 안내** (2026-05-13 외부 자문 권고로 MVP에서 제외)

이연한 기능을 PoC에 슬며시 끼워 넣고 싶다는 충동이 들면, 본 문서 0항(정체성 선언)을 다시 읽는다.

## 5. 변호사법 경계 (Bar Compliance Boundary)

변호사법 제109조는 비변호사의 대가성 있는 법률상담·법률관계 문서 작성·기타 법률사무 취급을 처벌한다(7년 이하 징역 또는 5천만원 이하 벌금).

대법원 2026.2.12 로폼 판결은 「생성형 AI를 활용해 이용자의 입력을 바탕으로 법률문서를 작성하는 경우 변호사법상 법률사무 취급에 해당할 수 있다」고 시사했다.

본 솔루션은 다음 경계 안에서만 작동한다.

- **합법 영역으로 설계되는 부분**: 사실 정보의 기계적 OCR·구조화, 자료 완결성 점수 산정, 공식 요금표 기반 산식 계산, 공개 절차 정보 정리, 빈칸형 템플릿 제공(사용자가 직접 채움).
- **명시 배제 영역**: 사안별 법률평가, 승패 판단, 소장·내용증명·민원 본문 작성, AI 사건검토 후 변호사 연결.

PoC 시작 전 화우 ESG센터 변호사법 사전 자문 의견서(출시 게이트 G5)를 확보한다. 의견서 없이 베타 출시 금지.

---

## 6. 분쟁 단계 처리 원칙

분쟁이 시작되는 순간 AI 역할은 종료된다.

1. 검산 결과 산식 차이가 발견된 경우에도 AI는 「분쟁 행동을 권유」하지 않는다.
2. 사용자가 분쟁 단계로 진입하기를 원하면 화우 ESG센터 pro bono/CSR, 공익변호사, 법률구조공단, 로스쿨 클리닉으로 **직접 연결**한다.
3. AI가 사건 검토를 한 뒤 변호사에게 넘기는 구조는 만들지 않는다.
